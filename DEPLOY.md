# Deploy TruthLens — GitHub + live link

This guide gets you a **public URL** you can share (e.g. `https://truthlens-frontend.onrender.com`).

**Stack:** GitHub (code) → Render (API + database + frontend)

Estimated time: **15–20 minutes**

---

## Prerequisites

- [GitHub](https://github.com) account
- [Render](https://render.com) account (free tier works)
- [GNews API key](https://gnews.io/) (free tier: 100 requests/day)

---

## Step 1 — Push to GitHub

Open PowerShell in the project folder:

```powershell
cd E:\TruthLens
git init
git add .
git commit -m "Initial commit: TruthLens news verifier"
```

Create a **new empty repository** on GitHub (no README). Then:

```powershell
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/truthlens.git
git push -u origin main
```

Replace `YOUR_USERNAME/truthlens` with your repo path.

---

## Step 2 — Deploy on Render (Blueprint)

1. Go to [dashboard.render.com](https://dashboard.render.com)
2. Click **New** → **Blueprint**
3. Connect your GitHub account and select the **truthlens** repository
4. Render reads `render.yaml` and creates:
   - `truthlens-db` — PostgreSQL
   - `truthlens-api` — Django API (Docker)
   - `truthlens-frontend` — React static site
5. When prompted for **GNEWS_API_KEY**, paste your key from [gnews.io](https://gnews.io/)
6. Click **Apply** and wait for all three services to finish deploying (first build can take **10–15 min** because of Whisper/ffmpeg)

### Your live URLs

After deploy succeeds:

| Service | URL |
|---------|-----|
| **App (share this)** | Your Render **static site** URL (e.g. `https://truthlens-frontend-xxxx.onrender.com`) |
| API | Your Render **web service** URL (e.g. `https://truthlens-api-lwta.onrender.com`) |
| Health check | `{API_URL}/api/health/` |

### Confirm the correct UI is deployed

The real TruthLens UI has **three tabs: Image · Video · Text** and **no “Show history”** button.

1. Open the frontend URL — footer should say **Build 1.1.0**
2. If you see “Show history” or a different layout, the static site is wrong:
   - **Root directory** must be `frontend`
   - **Publish directory** must be `dist`
   - Set `VITE_API_BASE_URL` = your API URL (no trailing slash)
   - **Manual Deploy** → Clear build cache & deploy

---

## Step 3 — Verify it works

1. Open the **frontend** URL in a browser
2. Paste a **text claim** and run verification (fastest test)
3. Check API health: `https://truthlens-api.onrender.com/api/health/` → `{"status":"ok"}`

### Free tier notes

- Services **spin down after ~15 min idle**; first request may take 30–60 seconds
- **Text claims** and **image OCR** work in production
- **Video upload** is disabled on Render (Whisper/PyTorch is too large for free-tier builds). Use text or a screenshot instead.
- Uploaded files are stored on the container disk (ephemeral on redeploy)

### If deploy failed before

Common causes we fixed in the repo:

1. **Docker build timeout** — removed Whisper from production image (`requirements-prod.txt`)
2. **Database env** — `DB_PASSWORD` no longer required when `DATABASE_URL` is set
3. **SSL** — `DB_SSL_REQUIRE=false` for Render PostgreSQL

## Optional — Frontend on Vercel instead

If you prefer Vercel for the UI:

1. Import the repo on [vercel.com](https://vercel.com)
2. Set **Root Directory** to `frontend`
3. Add environment variable:
   - `VITE_API_BASE_URL` = `https://truthlens-api.onrender.com`
4. Deploy

Then update Render **truthlens-api** environment variables:

- `CORS_ALLOWED_ORIGINS` = your Vercel URL (e.g. `https://truthlens.vercel.app`)
- `CSRF_TRUSTED_ORIGINS` = same URL

Redeploy the API service.

---

## Environment variables (API)

| Variable | Required | Description |
|----------|----------|-------------|
| `SECRET_KEY` | Yes | Auto-generated by Render |
| `DATABASE_URL` | Yes | Auto-linked from PostgreSQL |
| `GNEWS_API_KEY` | Yes | News search |
| `ALLOWED_HOSTS` | Yes | `truthlens-api.onrender.com` (in blueprint) |
| `CORS_ALLOWED_ORIGINS` | Yes | Frontend URL with `https://` |
| `DJANGO_SETTINGS_MODULE` | Yes | `config.settings.production` |
| `GOOGLE_FACTCHECK_API_KEY` | No | Optional extra source |

---

## Troubleshooting

| Problem | Fix |
|---------|-----|
| CORS error in browser | Ensure `CORS_ALLOWED_ORIGINS` matches your frontend URL exactly (https, no trailing slash) |
| 502 / slow first load | Free tier cold start — wait 60s and retry |
| `NO_SOURCES_FOUND` | Set `GNEWS_API_KEY` on the API service and redeploy |
| Build fails on Whisper | First deploy needs patience; or set `WHISPER_MODEL_SIZE=tiny` (already in blueprint) |
| OCR fails | Tesseract is in Docker; test with **text** first |
